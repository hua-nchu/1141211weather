# Change Proposal: 增強 Streamlit 應用的視覺化設計

## Why this change?

目前的 Streamlit 應用功能完整，但視覺呈現較為簡單，僅使用基本的表格展示天氣資料。參考中央氣象局官方網站 (https://www.cwa.gov.tw/V8/C/W/OBS_Temp.html) 的設計風格，我們希望提升使用者體驗，提供更直觀、美觀的資料視覺化。

**當前問題：**
- 資料僅以表格形式呈現，缺乏視覺吸引力
- 溫度變化不夠直觀
- 缺少圖表化的資料分析
- 整體設計風格較為單調

**預期改善：**
- 採用 CWA 官網的藍白色調設計
- 增加溫度卡片視覺化（類似氣象局的溫度顯示）
- 加入溫度分布圖表
- 提升整體 UI/UX 設計質感

## What changes?

### 視覺設計變更
1. **色彩主題**：採用 CWA 官網風格
   - 主色調：藍色系 (#1E88E5, #0D47A1)
   - 輔助色：白色、淺灰色
   - 強調色：橙色/紅色（高溫）、藍色（低溫）

2. **新增視覺化元件**
   - 溫度卡片：以卡片形式展示各地區溫度
   - 溫度條形圖：比較各地區溫度差異
   - 溫度折線圖：顯示歷史批次的溫度變化趨勢

3. **改善現有元件**
   - 統計資訊卡片增加圖標和顏色區分
   - 資料表格添加溫度條件格式（顏色標示）
   - 增加響應式設計

### 技術變更
1. **新增依賴套件**
   - plotly：互動式圖表
   - streamlit 原生圖表功能

2. **修改檔案**
   - `app.py`：主要 UI 改造
   - `requirements.txt`：新增依賴
   - 新增 CSS 樣式（如需要）

## Impact

### 用戶影響
- **正面**：更好的視覺體驗、更直觀的資料呈現
- **風險**：無，向下兼容，不影響現有功能

### 技術影響
- **依賴變更**：新增 plotly 套件
- **性能影響**：圖表渲染可能略增加載時間，但影響不大
- **維護性**：代碼結構保持清晰，易於維護

### 測試需求
- UI 視覺驗證（截圖比對）
- 功能完整性測試
- 不同資料量下的渲染測試
